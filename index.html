String.prototype.charCodeAt()
Description Parameters Return Value Examples Browser Compatibility
Description

The charCodeAt() method returns an integer between 0 and 65535 representing the UTF-16 code unit at the given index.

This method is part of the String prototype and is commonly used when you need to work with character encodings.

If you need to get the entire code point of a character, use String.prototype.codePointAt().

str.charCodeAt(index)
  1. Returns a number representing the UTF-16 code unit value
  2. Returns NaN if index is out of range
  3. Commonly used in string manipulation and encoding
Parameters

The method accepts a single parameter: index

Index is an integer between 0 and str.length - 1. If index is not a number, it defaults to 0.

If no index is provided, the default value is 0.

let str = "ABC"; str.charCodeAt(0); // returns 65 (ASCII value of 'A') str.charCodeAt(1); // returns 66 (ASCII value of 'B')
Return Value

Returns a number representing the UTF-16 code unit value of the character at the specified index.

The return value will be between 0 and 65535.

If the index is out of range, charCodeAt() returns NaN.

let str = "Hello";
  1. str.charCodeAt(0) returns 72 ('H')
  2. str.charCodeAt(999) returns NaN (out of range)

Browser Compatibility

The charCodeAt() method is supported in all modern browsers.

It has been part of the ECMAScript standard since version 1.

No polyfill is needed for this method.
